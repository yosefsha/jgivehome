Since there's no UI yet, you can verify sections 2 & 3 landed correctly in production a couple of ways:

1. Check the deploy logs (easiest — no extra tools needed) Our db/seeds.rb ends with puts "Seeded #{Campaign.count} campaigns, ...", and we wired db:seed into bin/docker-entrypoint so it runs on every boot. Open the jgivehome service → Logs, and look for a line like:

Seeded 2 campaigns, 8 donation options, 10 donations.

If you see that (with no errors above it from db:prepare), it confirms: migrations ran (the campaigns/donation_options/donations tables exist) and the seed data loaded successfully.

2. Query the production database directly with psql Render's managed Postgres gives you a connection string for external access: 1. Open the jgivehome-db database in the dashboard. 2. Go to Connect → copy the External Connection String (or PSQL Command). 3. Run it from your terminal (you have psql locally since you have Postgres installed): bash psql "<connection string from Render>" 4. Then query directly: sql SELECT title, goal_amount, slug FROM campaigns; SELECT count(*) FROM donation_options; SELECT donor_name, amount, status FROM donations LIMIT 10;

Either approach confirms the data is really there in production — option 1 is the quick check, option 2 lets you poke around freely. Want to try the logs first and paste what you see?

end
